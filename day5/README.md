# Day 5 - AWS Auto Scaling

## Objective
To create AWS Auto Scaling and automatically manage EC2 instances.

## Steps Performed
1. Created a Launch Template
2. Created an Auto Scaling Group
3. Configured VPC and Subnets
4. Set Min, Desired and Max capacity
5. Configured scaling policy
6. Created and launched instances

## Configuration
- Instance Type: t3.micro
- Minimum Capacity: 1
- Desired Capacity: 2
- Maximum Capacity: 2

## Result
Successfully created AWS Auto Scaling and launched 2 EC2 instances.
